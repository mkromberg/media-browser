Pro tips

Pro tip

The pro-tip slides from the Coding with Agents workshop, on one page. The short version of working well with an agent: plan before you prompt, keep it in bounds, and give it ways to check its own work. Several of these are the principles the kit's PROCESS.md is built on.

Mindset & judgement

Vibe coding is not agentic engineering.

Working with agents seriously is a disciplined practice, not casual improvisation.

simonwillison.net · what is agentic engineering

Freestyle prompting is for steering, not design.

Use ad-hoc prompts to nudge and course-correct; do the actual design through a plan you can review.

Don't ask Claude to "do something". Ask it to draft a plan for how to do it.

Turn a vague instruction into a plan you read and approve before any code is written.

Manage the context

Compact at logical milestones.

Condense the conversation at natural stopping points, not at random, so the context stays focused.

/compact

Permissions & modes

Start in yolo, but run in auto.

Launch fast with --dangerously-skip-permissions, then keep everyday work in the safer auto mode.

Use plan mode when scoping.

Let Claude research and propose read-only, before it touches the code. Shift-Tab into plan mode.

Build in correctness

Give Claude a way to verify its own work.

Tests and checks let the agent confirm its output instead of guessing.

Strong types REALLY helps.

Static types are guardrails: they catch the agent's mistakes before you have to.

Use the static analysers, linters, and formatters your tool chain provides.

Every automated correctness signal your stack offers is one more check the agent gets for free.

Where humans add value

Review designs, architecture, APIs, test surfaces... not every line of code.

Spend your attention where judgement matters, not on scrutinising every generated line.
