# PR notes: Issue #17 - Static-asset fallback in Get

Plan: https://github.com/mkromberg/media-browser/blob/main/docs/plans/original-plan.md#L110

Part of epic #6 (Browser frontend).

## Scope

`ServeStaticAsset frontendRoot urlPath` and the `Get` dispatch fallback: any endpoint other than `/files` is served as a static asset from `apl-service/frontend/` (via `AppState.FrontendRoot`), or a 404 if `ServeStaticAsset` finds nothing.

`Initialize` grows one line to compute `AppState.FrontendRoot` (`jarvis.CodeSource,'/../frontend'`, the sibling of `APLSource` set once at startup) alongside the existing `AppState.RootFolder`, since `Get` needs it on every request and has no other source for it - it isn't in the issue's own scope line, but `Get`'s fallback cannot work without it.

`index.html`/`app.js`/`style.css` themselves are out of scope (issue #18/#19); `apl-service/frontend/` still holds only `.gitkeep` after this issue. Coverage instead uses a fixture tree, `tests/fixtures/frontend/`.

## Acceptance criteria

`GET /` returns 200 and serves `index.html`; `GET /bogus` returns 404.

## Cycle 1: RED

### Design notes

`req.Endpoint` arrives already URL-decoded by Jarvis (confirmed by reading `vendor/jarvis/Jarvis.dyalog`: `Endpoint←URLDecode Endpoint` runs before any application code sees it), so a literal `'..'` path-segment check in `ServeStaticAsset` is not bypassable via percent-encoding.

`ServeStaticAsset` rejects any `urlPath` containing a `'..'` segment outright, rather than canonicalising and prefix-checking the way the (separate, epic-3) `ResolveSubfolder` will for user-supplied folder names. The frontend tree ships as part of the service itself, is never user-writable, and contains no symlinks, so the stronger real-path canonicalisation that a user-controlled photo root needs is not a proportionate response here; a segment-level `'..'` deny check is simpler and sufficient for a fixed, self-contained asset tree. `Test_ServeStaticAssetRejectsDotDotSegmentEvenWhenTargetExists` and `...RejectsDotDotSegmentBuriedInADeeperPath` prove the check actually blocks traversal (against a real file one level above the fixture root), not just that the escaped target happens to be missing.

Only `.html`/`.css`/`.js` get exact content types (matching the three files the epic actually ships); anything else present under `frontendRoot` still serves as `application/octet-stream` rather than being rejected, since rejecting-by-extension is `ServeFile`'s (epic-4) job for arbitrary user media, not this fixed asset tree's.

`ServeStaticAsset` returns `⍬` uniformly for a rejected path, a missing file, or a path naming a directory rather than a regular file - `Get` collapses all three to a 404, matching the acceptance criterion's `GET /bogus` case and giving a traversal attempt no distinguishable response from an ordinary 404.

### Test surface

`apl-service/tests/unit/ServeStaticAssetTests.aplf` (8 tests) drives `ServeStaticAsset` directly against a new fixture tree, `apl-service/tests/fixtures/frontend/` (`index.html`, `app.js`, `style.css`, `data.xyz` for the unrecognised-extension case, an empty `sub/` for the directory case) plus a sibling `apl-service/tests/fixtures/secret-outside.txt` used only to prove the traversal guard against a real, readable, existing file.

`apl-service/tests/unit/GetTests.aplf` gains two cases and one changed fixture (`FrontendFixture`, wired into every existing case's `AppState` alongside `RootFolder`, since the fallback branch now runs for any non-`/files` endpoint including `Test_GetSets404ForUnknownEndpoint`'s `/nonsense`). `StubReq` grows a `.SetHeader` stub recording onto `.Response.ContentType`, alongside the existing `.Fail`.

`apl-service/tests/unit/InitializeTests.aplf` gains one case for the new `AppState.FrontendRoot` assignment.

### Anticipated implementation

- `ServeStaticAsset frontendRoot urlPath`: reject if any `'/'`-split segment of `urlPath` is `'..'`; map `'/'` to `frontendRoot,'/index.html'`, else `frontendRoot,urlPath`; check `⎕NEXISTS` then `2=1 ⎕NINFO` (mirroring `CheckFolder`'s existence-before-type ordering, since `⎕NINFO` on a nonexistent path errors rather than returning a sentinel); read text content via `⊃⎕NGET file 1` (the same idiom `ScaffoldTests` already uses); derive content type from the `⎕NPARTS`-extracted, lower-cased extension via a three-entry table (`html`/`css`/`js`) defaulting to `application/octet-stream`. Returns `⍬` on any rejection, or a namespace with `.Body`/`.ContentType`.
- `Get`: for any `req.Endpoint` other than `'/files'`, call `ServeStaticAsset AppState.FrontendRoot req.Endpoint`; on `⍬`, `req.Fail 404`; otherwise `'Content-Type' req.SetHeader asset.ContentType`, assign `req.Response.Payload`, and return the body.
- `Initialize`: one additional assignment, `AppState.FrontendRoot←jarvis.CodeSource,'/../frontend'`, alongside the existing `RootFolder` line.
