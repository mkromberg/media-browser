# PR notes: Issue #8 - CheckFolder, ScanFolder, ClassifyExtension, FormatTimestamp

Plan: https://github.com/mkromberg/media-browser/blob/main/docs/plans/original-plan.md#L128-L133

Part of epic #1 (Bootstrap and root listing).

## Scope

Four pure, unit-testable folder-scanning primitives:

- `ClassifyExtension ext`: lower-cases, strips a leading `.`, and maps to `'image'`, `'video'`, or `''`.
- `CheckFolder folder`: classifies an absolute path as `'ok'`, `'not-found'`, or `'not-a-directory'`.
- `ScanFolder folder`: one wildcard `⎕NINFO` call over the folder's immediate children, split into `(paths sizes mtimes subfolders)`, no recursion.
- `FormatTimestamp ts`: a `⎕TS`-style vector to `YYYY-MM-DDTHH:MM:SS`.

Readability is not a `CheckFolder` outcome. An unreadable folder surfaces later as a scan error that `Get`'s `:Trap 0` turns into a 500, so a separate `'not-readable'` result (and the awkward `chmod 000` fixture it required) is dropped. The plan is updated to match.

## Open question confirmed empirically

The plan flagged whether `⎕NINFO` modification times are local or UTC. Checked with a live `dyalogscript` run on this environment (Dyalog 20.0, Linux, `TZ=Europe/London`, currently BST = UTC+1): a file created at 13:50 local reports mtime `2026 7 9 13 50 46 ...`, matching `⎕TS` (13:51) rather than the UTC 12:50. So `FormatTimestamp` emits no trailing `Z`, and `ScanFolderTests.Test_ScanFolderMtimesAreLocalWallClock` locks this by asserting a just-created file's mtime equals the local wall clock captured either side of the scan. The `11 ⎕NINFO` readability question is moot now that readability is not a `CheckFolder` outcome.

Two mechanical facts settled the implementation shape: `⎕IO` is `1` here, so `ScanFolder` reads the `⎕NINFO` result by IO-independent vector destructuring rather than indexed pick; and `819⌶` (lower-case) is withdrawn in favour of `⎕C`.

### Test surface

Four namespace scripts under `apl-service/tests/unit/` and a committed fixture tree under `apl-service/tests/fixtures/scan/` (`a.JPG` 1 byte, `b.mp4` 2 bytes, `notes.txt` 3 bytes, and a `sub/` subfolder). `CheckFolderTests` reuses that tree for its `'ok'` and `'not-a-directory'` cases. The empty-folder and mtime cases, which cannot be represented in git, are created with `⎕MKDIR` and removed with `3 ⎕NDELETE` (recursive); no shelling out.

### Anticipated implementation

- `ClassifyExtension`: `⎕C` then strip a leading `.`, membership against the two extension lists.
- `CheckFolder`: `⎕NEXISTS`, then `1 ⎕NINFO` for type.
- `ScanFolder`: `(0 1 2 3)(⎕NINFO⍠1)folder,'/*'`, destructured; filter type 2 for files, type 1 for subfolders, bare names via `∊1↓⎕NPARTS`.
- `FormatTimestamp`: zero-pad the first six fields of the vector and join.
