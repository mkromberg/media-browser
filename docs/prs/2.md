## Cycle 1: RED

### Test surface

`apl-service/tests/unit/ScaffoldTests.aplf` (namespace script, seven `Test_*` functions) plus a fixture, `apl-service/tests/fixtures/orphan_jarvis_config/jarvis.config.json`, and the harness that runs them, `apl-service/tests/run-tests.apls`.

The harness fixes `APLSource/*.aplf` and `tests/unit/*.aplf` with one helper, `Fix←{2 ⎕FIX'file://',⍵}`. `2 ⎕FIX` fixes "a series of named namespaces or function definitions, or a combination" from a script file, so the same call handles a bare function (`APLSource/*.aplf`) and a namespace script bundling several functions (`tests/unit/*.aplf`) - default `⎕FIX` (`X=1`) rejects a bare tradfn, which is the only reason an earlier draft split this into two helpers. `2 ⎕FIX` returns the established name(s) as strings rather than a reference, so the harness does `⍎name` to get a reference where it needs one (test namespaces), and nothing further for source files, where the function only needs to be established under its own name. The harness then discovers niladic `Test_*` functions per namespace via `⎕NL 3`, runs each under `:Trap 0`, and exits 0/1 on aggregate pass/fail.

`run-tests.apls` (not `.aplf`): it is a file dyalogscript executes directly as top-level statements and never fixes, as distinct from `.aplf`, which now names only a file `⎕FIX`ed into the workspace. The plan is updated accordingly, including for the two other script files later issues will add (`scripts/start-server.apls`, `tests/fixtures/generate_fixtures.apls`).

### Harness fixes made during GREEN

Two issues surfaced only once real assertions ran against real content, both fixed in `run-tests.apls` and `ScaffoldTests.aplf` rather than in the feature code:

- `⎕OFF` did not propagate a computed exit code (`⎕OFF ×failures`) reliably; dyalogscript only honours a literal or pre-bound variable argument. Fixed by assigning to a variable first (`exitCode←×failures ⋄ ⎕OFF exitCode`).
- `⎕NC` always returns a vector, even for a single name (`R←⎕NC Y` is documented as "a simple numeric vector containing one element per name in Y"). Comparing that against a scalar with `=` yields a one-element boolean vector, not a scalar, so `Test_AppConfigHasRootFolder` and `Test_VendoredJarvisFixes` returned a truthy-but-non-scalar result that failed a strict `≡1` check. Fixed at the call sites with `⊃⎕NC...`, and hardened the harness's own pass/fail comparison to `(,1)≡,ok` so a future test making the same mistake reports its real result instead of a spurious failure.
